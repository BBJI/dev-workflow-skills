# 无障碍合规参考

设计规范中的无障碍标准和合规要求详细说明。

## WCAG 2.2 合规级别

### A级（最低）
- 1.1.1 非文本内容：图片有alt文本，输入有标签
- 1.3.1 信息与关系：语义化HTML，ARIA地标
- 1.4.1 颜色使用：颜色不是唯一指示器
- 2.1.1 键盘：所有功能可通过键盘操作
- 2.4.1 绕过区块：跳过导航链接
- 2.4.3 焦点顺序：逻辑Tab序列
- 3.1.1 页面语言：lang属性
- 4.1.1 解析：有效标记
- 4.1.2 名称、角色、值：自定义组件的ARIA

### AA级（标准目标）
- 1.4.3 对比度（最低）：正文4.5:1，大字3:1
- 1.4.11 非文本对比度：UI组件3:1
- 2.4.6 标题和标签：描述性
- 2.4.7 焦点可见：可见的焦点指示器
- 3.2.3 一致导航：跨页面顺序相同
- 3.3.1 错误识别：清晰的错误信息
- 3.3.2 标签或说明：输入标注

### AAA级（增强，可选）
- 1.4.6 对比度（增强）：正文7:1，大字4.5:1
- 2.4.9 链接目的（仅链接文本）：仅凭链接文本可理解
- 3.1.4 缩写：展开机制

## 组件特定无障碍规格

### 按钮
- 角色：`button`（原生）或 `role="button"`（自定义）
- 无障碍名称：可见文本或 `aria-label`
- 键盘：Enter和Space激活
- 禁用状态：`aria-disabled="true"`，不仅仅是视觉变化
- 纯图标按钮：必须提供 `aria-label`

### 表单输入
- 每个输入框有关联的 `<label>` 或 `aria-label`
- 必填字段：`aria-required="true"` + 视觉指示器
- 错误信息：通过 `aria-describedby` 关联错误文本
- 分组相关输入：`<fieldset>` + `<legend>`
- 自动填充：常用字段设置 `autocomplete` 属性

### 模态框/对话框
- `role="dialog"` 和 `aria-modal="true"`
- `aria-labelledby` 指向对话框标题
- 焦点锁定：Tab仅在对话框内循环
- Escape键关闭对话框
- 关闭后焦点返回触发元素
- 背景滚动锁定

### 数据表格
- `<th>` 带有 `scope` 属性
- caption元素用于表格描述
- 复杂表格：使用 `headers` 属性关联
- 可排序列：`aria-sort` 属性
- 分页：导航上设置 `aria-label`

### 导航
- 地标：`<nav>` 带 `aria-label`（多个导航时）
- 当前页：`aria-current="page"`
- 跳转链接：第一个可聚焦元素，链接到主内容
- 移动端菜单：切换按钮设 `aria-expanded`，菜单设 `aria-controls`

### 标签页
- `role="tablist"`、`role="tab"`、`role="tabpanel"`
- 激活标签设 `aria-selected`
- 标签页设 `aria-controls` 指向面板
- 方向键在标签之间移动
- Tab键将焦点移到面板内容

## 屏幕阅读器测试要点

至少使用以下组合测试：
- **VoiceOver**（macOS/iOS）：Safari
- **NVDA**（Windows）：Firefox
- **TalkBack**（Android）：Chrome

需要验证的关键事项：
1. 页面标题在加载时正确播报
2. 标题结构可导航（H1 → H2 → H3）
3. 表单可在无视觉情况下完成
4. 动态更新被播报（aria-live）
5. 自定义组件具有正确的角色和状态

## 动画与运动无障碍

- 尊重 `prefers-reduced-motion: reduce`
- 启用减少运动时：
  - 用即时状态变化替代动画
  - 移除视差效果
  - 用手动导航替代自动轮播
  - 保留必要的运动（加载指示器）但简化
- 绝不自动播放视频或音频
- 为任何自动更新内容提供暂停/停止控制

## 颜色与对比度

### 最低对比度比率（WCAG AA）
| 元素 | 比率 | 示例 |
|------|------|------|
| 正文（< 24px） | 4.5:1 | #333333 on #FFFFFF |
| 大字（≥ 24px 或 ≥ 18.66px粗体） | 3:1 | #767676 on #FFFFFF |
| UI组件和图形对象 | 3:1 | 按钮边框、图标 |
| 焦点指示器 | 3:1 | 与相邻颜色的对比 |

### 色盲考量
- 不要仅依赖红/绿区分
- 在颜色之外使用图案、图标或文本标签
- 使用 Coblis 或类似色盲模拟器测试
- 常见类型：红色盲（Protanopia）、绿色盲（Deuteranopia）、蓝色盲（Tritanopia）
